📊 Monitoring & Troubleshooting

Level: Intermediate

Monitoring adalah proses memantau kesehatan jaringan, sedangkan Troubleshooting adalah langkah-langkah untuk menemukan dan memperbaiki masalah. Bayangkan seperti dokter: monitoring = pemeriksaan rutin, troubleshooting = pengobatan saat sakit.

📖 Penjelasan Singkat

Monitoring

  • Menggunakan tools seperti ping, traceroute, netstat, atau aplikasi seperti Wireshark, Nagios, Zabbix.
  • Memantau bandwidth, latency, packet loss, uptime.
  • Mendeteksi masalah sebelum berdampak besar.

Troubleshooting

  • Mengidentifikasi masalah (apakah di perangkat, kabel, router, atau server).
  • Menggunakan metode sistematis: Identify → Isolate → Fix → Verify.
  • Contoh: jika tidak bisa akses website, cek koneksi lokal → gateway → DNS → server tujuan.

Monitoring menjaga jaringan tetap sehat, troubleshooting menyelesaikan masalah saat terjadi gangguan.

🛠️ Praktik Kecil

Coba lakukan traceroute untuk melihat jalur data ke server tujuan:

# Windows (Command Prompt)
tracert google.com

# Linux/Mac (Terminal)
traceroute google.com

👉 Dari output ini, kamu bisa melihat hop demi hop (router yang dilewati) hingga sampai ke tujuan.

🎮 Mini Challenge

Kamu diminta memperbaiki jaringan kantor. Gejalanya: komputer bisa ping gateway, tapi tidak bisa membuka website.

Tugas: Buat langkah troubleshooting sistematis untuk menemukan penyebab masalah. Sebutkan tools apa saja yang akan kamu gunakan di setiap langkah.
