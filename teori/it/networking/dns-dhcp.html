🌍 DNS & DHCP

Level: Beginner

DNS (Domain Name System) adalah layanan yang menerjemahkan nama domain (seperti google.com) menjadi alamat IP. DHCP (Dynamic Host Configuration Protocol) adalah layanan yang memberikan alamat IP otomatis ke perangkat. Bayangkan DNS seperti buku telepon yang menghubungkan nama dengan nomor, dan DHCP seperti resepsionis yang langsung memberikan nomor kamar ke tamu hotel baru.

📖 Penjelasan Singkat

  • DNS: mempermudah manusia mengingat alamat website tanpa harus hafal angka IP.
  • DHCP: menghemat waktu admin jaringan karena perangkat mendapat IP otomatis.
  • Tanpa DNS: kita harus mengetik IP (contoh: 142.250.190.78) untuk membuka Google.
  • Tanpa DHCP: setiap perangkat harus diatur manual alamat IP-nya.

Keduanya bekerja sama agar koneksi jaringan lebih mudah, cepat, dan efisien.

🛠️ Praktik Kecil

Coba cek alamat DNS dan DHCP yang digunakan perangkatmu:

# Windows (Command Prompt)
ipconfig /all

# Linux/Mac (Terminal)
cat /etc/resolv.conf   # lihat DNS
nmcli dev show | grep DHCP4   # lihat DHCP

👉 Dari sini kamu bisa tahu server DNS yang dipakai dan apakah IP diberikan oleh DHCP.

🎮 Mini Challenge

Bayangkan kamu mengelola warnet dengan 20 komputer.

Tugas: Jelaskan kenapa lebih praktis menggunakan DHCP daripada mengatur IP secara manual di setiap komputer. Dan apa risiko jika server DNS internal warnetmu mati?
