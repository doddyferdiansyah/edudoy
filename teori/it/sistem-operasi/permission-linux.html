ğŸ” Permission & User Management

Level: Intermediate

Setiap sistem operasi harus bisa mengatur siapa yang boleh melakukan apa. Permission adalah mekanisme keamanan dasar yang memastikan file dan folder tidak bisa diakses sembarangan. Bayangkan komputer seperti perpustakaan digital: ada buku umum yang bisa dibaca semua orang, ada juga arsip khusus yang hanya boleh diakses pustakawan.

ğŸ“– Penjelasan Singkat

  • User: identitas individu yang menggunakan sistem.
  • Group: kumpulan user dengan hak akses yang sama.
  • Permission: hak akses file/folder (read, write, execute).
  • Ownership: setiap file punya pemilik (user) dan group.

Dengan permission, OS bisa mencegah user biasa menghapus file sistem atau mengakses data orang lain.

Coba lihat permission file di Linux:

ls -l

Output contoh:

-rw-r--r--  1 doddy users   1200 Oct 29 20:00 catatan.txt

Ubah permission agar hanya pemilik yang bisa membaca file:

chmod 600 catatan.txt
