💾 Memory Management

Level: Intermediate

RAM adalah ruang kerja sementara bagi komputer. Sistem Operasi bertugas mengatur siapa yang boleh menggunakan memori, kapan, dan seberapa banyak. Bayangkan RAM seperti meja kerja: semakin besar meja, semakin banyak pekerjaan bisa dilakukan sekaligus. OS adalah manajer kantor yang memastikan meja dipakai secara adil dan efisien.

📖 Penjelasan Singkat

  • Virtual Memory: teknik agar komputer bisa “meminjam” ruang dari harddisk ketika RAM penuh.
  • Paging & Segmentation: cara OS membagi memori menjadi blok-blok kecil agar lebih mudah dikelola.
  • Swap: area di disk yang digunakan sebagai cadangan RAM.
  • Memory Protection: mencegah satu proses mengganggu memori proses lain.

Tanpa manajemen memori, aplikasi bisa saling berebut RAM dan menyebabkan sistem crash.

🛠️ Praktik Kecil

Coba cek penggunaan RAM di sistem kamu:

# Windows (Command Prompt / PowerShell)
systeminfo | findstr /C:"Total Physical Memory"

# Linux/Mac (Terminal)
free -h

👉 Dari sini kamu bisa melihat total RAM, penggunaan saat ini, dan swap memory.

🎮 Mini Challenge

Bayangkan kamu punya meja kerja kecil, tapi harus mengerjakan banyak tugas sekaligus. Kamu mulai menaruh sebagian kertas di lantai (swap). Akibatnya, pekerjaan jadi lebih lambat.

Tugas: Jelaskan kenapa komputer dengan RAM kecil tapi swap besar tetap terasa lambat, meskipun secara teori punya “memori tambahan”.
