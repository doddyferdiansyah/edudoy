Level: Intermediate

Packet analysis adalah teknik menganalisis lalu lintas jaringan untuk memahami bagaimana data bergerak, mendeteksi anomali, atau menginvestigasi serangan. Bayangkan seperti membuka amplop surat di kantor pos untuk melihat isi pesanâ€”bedanya, di dunia digital kita hanya mengamati salinan data, bukan merusak komunikasi.

ğŸ“– Apa Itu Packet Analysis?

  • Proses menangkap dan memeriksa paket data yang lewat di jaringan.
  • Digunakan untuk troubleshooting, monitoring, dan investigasi keamanan.
  • Tools populer: Wireshark, tcpdump.

ğŸ“Š Informasi yang Bisa Dilihat

  • Alamat IP sumber & tujuan.
  • Port yang digunakan (HTTP, HTTPS, DNS, dll).
  • Protokol komunikasi (TCP, UDP, ICMP).
  • Payload (isi data, jika tidak terenkripsi).

  • Wireshark: seperti CCTV lalu lintas yang merekam semua kendaraan di jalan raya.
  • tcpdump: seperti petugas yang mencatat nomor polisi kendaraan yang lewat.

ğŸ› ï¸ Praktik Kecil dengan Wireshark

  1. Install Wireshark di laptopmu.
  2. Buka Wireshark dan pilih interface jaringan (misalnya Wi-Fi).
  3. Mulai capture, lalu buka browser dan akses example.com.
  4. Stop capture, lalu filter dengan http atau dns.
  5. Amati: alamat IP tujuan, request DNS, dan paket HTTP.

ğŸ› ï¸ Praktik Kecil dengan tcpdump

  1. Buka terminal (Linux/Mac).
  2. Jalankan: sudo tcpdump -i wlan0 -c 10 (ganti wlan0 dengan interface-mu).
  3. Lihat 10 paket pertama yang lewat.
  4. Diskusikan: protokol apa yang terlihat? IP mana yang sering muncul?

Buat investigasi sederhana dengan packet analysis:

  • Lakukan capture saat browsing ke sebuah website.

  • Bandingkan hasil saat mengakses website HTTP vs HTTPS.
  • Diskusikan: apa perbedaan data yang terlihat?

Tugas tambahan: Buat laporan singkat (1 halaman) berisi hasil analisis paketmu, lengkap dengan screenshot Wireshark/tcpdump.
